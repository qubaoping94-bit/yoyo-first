# Workflows

## New Special-Topic Page

1. Read workspace instructions and the relevant Obsidian memory index.
2. Study the user's reference images and title.
3. Define the slug, short title, tags, use case, and central thesis.
4. Generate or revise the master image.
5. Save final assets under `Desktop/three/output/<slug>/`.
6. Create `index.html` and a Chinese-title redirect page.
7. Append one entry to `scripts/product-evolution-pages.config.json`.
8. Update `scripts/site-directory-common.cjs` and `scripts/build-wechat-share-poster.cjs`.
9. Run build scripts and local QA.
10. Deploy the prepared public folders to Cloudflare Pages.
11. Run direct online checks and audit scripts.
12. Update memory and report the public URL.

## Targeted Image Revision

1. Identify the exact region the user wants changed.
2. Preserve all unrelated scenes, labels, and supporting illustrations.
3. Regenerate or edit only the requested section.
4. If the new output drifts too far from the accepted image, regenerate with stricter preservation language.
5. Continue to the page/build/deploy chain after user acceptance or when the user says to continue.

## Page Acceptance

A page is not complete until the public route, public PNG URL, product-evolution hub, site list, WeChat assets, and audit scripts all agree.
