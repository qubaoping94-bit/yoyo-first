# Security

## Secrets

Do not copy API keys, cookies, tokens, passwords, browser session files, or private account data into the skill package.

## Git Safety

- Clone the target repository into a clean upload directory before publishing.
- Preserve unrelated remote files.
- Do not force-push.
- Do not use destructive git commands unless the user explicitly requests them.

## Claim Safety

Do not invent compliance data, test results, lifetime years, standards, patent counts, or percentage improvements. Use conservative wording when evidence is not supplied.

## Cloud Deployment

Use the existing Cloudflare Pages project names and audit scripts. Treat historical green checks as routing evidence only; rerun current checks before claiming success.

## Local Memory

Only save durable workflow facts, stable paths, decisions, and open loops to Obsidian memory. Do not save full chat logs or sensitive platform configuration.
