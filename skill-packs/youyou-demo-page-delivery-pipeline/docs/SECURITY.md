# 安全说明

## 不应提交的内容

- `.env`、token、cookie、API key、验证码、浏览器会话文件。
- 客户隐私、个人联系方式、付款信息、身份证号等敏感数据。
- 与 skill 无关的项目源码、图片素材或部署凭据。

## Git 和部署安全

- 发布前使用干净 clone，不覆盖远端已有无关文件。
- 不使用强制推送或破坏性 Git 命令。
- 推送前检查 `git status`，确认只包含本 skill pack 相关文件。

## 内容安全边界

- 不写未经验证的检测数据和绝对承诺。
- 对健康、防火、抗菌、防霉、隔音等功能只使用稳妥表达。
- 需要数值或认证时，必须由用户提供对应检测报告或项目资料。
