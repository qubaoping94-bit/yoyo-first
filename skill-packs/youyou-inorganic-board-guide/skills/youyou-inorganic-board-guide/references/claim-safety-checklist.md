# Claim Safety Checklist

Use this before answering technical, price, construction, durability, testing, or standards questions.

## Evidence Levels

| Level | What it means | How to answer |
|---|---|---|
| Public link evidence | A public Youyou page exists for the topic | Explain the topic and link the page. |
| User-supplied evidence | The user supplied a report, quotation, image, or document in the current task | Quote or summarize only the supplied material. |
| No evidence in context | The user asks for exact values, standards, or promises without source material | State the boundary and ask for official report/project details. |

## Red-Line Claims

Do not invent or imply exact:

- fire rating or formal standard
- antibacterial rate
- waterproof duration
- deformation rate
- service life
- patent count
- national test number
- delivery date
- construction cost
- warranty promise
- "best in the industry" or "only one" claims

## Safe Alternatives

Use:

- `建议先看对应专项演示页`
- `最终以正式检测报告为准`
- `需要结合项目图纸、样板和报价条件确认`
- `更适合从材料逻辑和应用场景来理解`
- `可以重点关注...`

Avoid:

- `保证`
- `绝对`
- `永久`
- `100%`
- `行业唯一`
- `所有环境都适用`
- `不用看报告`

## Before Sending A Performance Answer

Check:

1. Did I include a matching public link?
2. Did I avoid unsupported exact numbers?
3. Did I say what needs official confirmation?
4. Did I give the customer a useful next action?

If any answer is no, revise before responding.
